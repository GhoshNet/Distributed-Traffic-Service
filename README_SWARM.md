# Oracle Cloud Docker Swarm Deployment Guide

This guide specifically details how to deploy the Distributed Traffic Service across 3 Oracle Cloud Always Free ARM instances to demonstrate high availability, fault tolerance, and multi-node container orchestration.

## Phase 1: Provisioning the Virtual Machines

1. Log into your **Oracle Cloud Console** and navigate to **Compute > Instances**.
2. Click **Create Instance**. You need to repeat this process to create **3** separate VMs (`manager-1`, `worker-1`, `worker-2`).
3. For each instance:
   - **Shape:** Click "Change Shape" -> Select "Ampere" -> Check the `VM.Standard.A1.Flex` shape. Allocate **1 OCPU** and **8 GB RAM** to each VM (This perfectly balances the 24GB free tier limit across 3 VMs).
   - **Image:** Ubuntu 22.04 or 24.04 (aarch64).
   - **Networking:** Keep them in the same VCN (Virtual Cloud Network) and Subnet.
   - **Placement:** (Crucial!) For `worker-1`, select *Fault Domain 2*. For `worker-2`, select *Fault Domain 3*. This proves datacenter-level redundancy.
   - Save your generated SSH keys! You will need them to log in.

## Phase 2: Opening the Firewalls (Security Lists)

To allow Docker Swarm to communicate, you must open specific ports in the Oracle Cloud firewall.
1. In the Oracle Cloud Console, navigate to **Networking > Virtual Cloud Networks**.
2. Click your VCN -> **Security Lists** -> Default Security List.
3. Add the following **Ingress Rules** (Source CIDR: `0.0.0.0/0` or ideally restricted to your VCN's `10.0.0.0/16` subset):
   - **TCP**: `2377` (Cluster management)
   - **TCP/UDP**: `7946` (Container network discovery)
   - **UDP**: `4789` (Overlay network traffic)
   - **TCP**: `8080`, `3000` (API Gateway and Frontend Web traffic for your demo)
4. SSH into **each** VM and open exactly the same ports on the local Ubuntu firewall:
   ```bash
   sudo iptables -I INPUT 6 -m state --state NEW -p tcp --dport 2377 -j ACCEPT
   sudo iptables -I INPUT 6 -m state --state NEW -p tcp --dport 7946 -j ACCEPT
   sudo iptables -I INPUT 6 -m state --state NEW -p udp --dport 7946 -j ACCEPT
   sudo iptables -I INPUT 6 -m state --state NEW -p udp --dport 4789 -j ACCEPT
   sudo iptables -I INPUT 6 -m state --state NEW -p tcp --dport 8080 -j ACCEPT
   sudo iptables -I INPUT 6 -m state --state NEW -p tcp --dport 3000 -j ACCEPT
   sudo netfilter-persistent save
   ```

## Phase 3: Installing Docker

On **all 3 VMs**, run the official Docker installation script:
```bash
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh
sudo usermod -aG docker ubuntu
```
*(You will need to log out and log back into your SSH session for the user group changes to take effect).*

## Phase 4: Initializing the Swarm

1. **On Node 1 (`manager-1`):**
   ```bash
   docker swarm init
   ```
   *This command will output a token string starting with `docker swarm join --token...`*

2. **On Node 2 and Node 3 (`worker-1`, `worker-2`):**
   Paste the exact command generated by the manager node into your worker terminals to attach them to the cluster.

## Phase 5: Deployment

1. Transfer your codebase to Node 1 (`manager-1`) via `scp` or `git clone`.
2. Navigate into the project directory.
3. Make the script executable:
   ```bash
   chmod +x deploy-swarm.sh
   ```
4. Run the script:
   ```bash
   ./deploy-swarm.sh
   ```

## Phase 6: System Verification

Once the script completes, run the following command on `manager-1`:
```bash
docker service ls
```
You should see all columns populated (e.g., `2/2` replicas for the application services). Wait ~2-3 minutes for the databases and RabbitMQ cluster to form.
Visit `http://<MANAGER_PUBLIC_IP>:3000` to interact with the frontend demo!
