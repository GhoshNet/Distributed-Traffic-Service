Imagine a world in which all road-vehicle drivers are required to prebook every journey that they wish to make, in particular, so that no driver is allowed to start a journey without having received a notification that the requested journey is acceptable. The exercise is to design, implement and demonstrate a system whose primary purpose is to allow drivers to book/cancel journeys and that is intended to be globally-accessible.

Note that the focus is not on how the system decides whether or not a journey is acceptable (but, of course, the system should recognize the fact that some requests may not be compatible) or how the requirement that journeys have been booked before starting is enforced (but should support such enforcement). The design should focus on providing appropriate levels of performance, scalability, availability and reliability, considering that the system may need to grow to be used by millions of users throughout the world.

There is no ‘right’ design for such a system. In particular, we should begin by specifying a reasonable set of requirements for the system taking into account the level of dependability that might be expected by users of this service. The design should then focus on how the system (and any individual subcomponents that have been identified) might be designed to satisfy the requirements that are specified. Be careful not to over engineer the system since providing higher levels of dependability is usually detrimental to the cost of operating/using a system.

Your system should be structured as a set of 'N' (distributed) software services that operate in a loosely-coupled manner, where N >= 6.
